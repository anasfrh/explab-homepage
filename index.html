Free, self-hosted experimentation on your own warehouse.

ExpLab is an open-source project for running experiment analysis directly on top of your own warehouse. It is meant for teams that want a simple self-hosted starting point without paying SaaS event fees or moving product data into a separate analytics vendor.

Why ExpLab

Three reasons to use this instead of a hosted experimentation vendor.

01

You keep the data path

Metrics and conversion events are queried directly from your own database, so sensitive product data does not need to be copied into another analytics tool.

02

You control the cost

The project is open source and free to run, which means no MTU pricing and no event-volume tax as usage grows.

03

You keep your own logic

Metric definitions, conversion logic, and experiment analysis stay close to the SQL your team already trusts instead of being forced into a vendor model.

What you get

A practical starting point for running and reading experiments.

1

Warehouse-native analysis

Run statistical analysis where the data already lives, with SQLite included for demo and local testing.

2

Statistical guardrails

Use primary, secondary, and guardrail metrics alongside SRM checks, balance diagnostics, and multiple-testing corrections.

3

Configurable metrics

Tune winsorization, override experiment settings, and inspect the raw SQL when you need a closer look.

4

Planning tools included

Use the power calculator and simulator to size experiments and test the project before wiring a live warehouse.

Why It Stays Free

Built with LLMs, released back to the people whose work made that possible.

ExpLab was built fully with LLMs using Codex and Gemini Pro. The project is free and open source because modern LLMs exist in large part thanks to the enormous body of open software, public writing, forums, mailing lists, and shared internet knowledge created by people over many years.

In a market full of SaaS-shaped, vibe-coded tools, it felt more honest to make something that gives back instead of closing it off. ExpLab started with one person, but it does not need to stay owned by one person.

If it is useful to you, share it, star it, fork it, and change it however you need. Run it for your team, reshape it for your stack, and take it somewhere better. The point is not just to use it, but to let more people own it.
